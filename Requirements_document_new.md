NGSIM-US-101 数据集处理规范文档
=====================================

一、概述
--------
本文档旨在规范NGSIM-US-101数据集的数据处理流程，明确各阶段处理要求、输出内容及质量标准，确保数据处理工作的系统性和可追溯性。

二、环境配置要求
----------------
1. 编程语言：Python
2. 虚拟环境：conda py-310
3. 代码存储路径：./code/“时间”/
4. 文档输出路径：./doc/“时间”/
5. 图片输出路径：./doc/“时间”/pic/

   注意：报告中引用图片时请使用相对路径 ./doc/pic/xxx.png

四、数据处理流程
----------------

4.1 轨迹平滑与去噪（Denoising）
-------------------------------
处理目标：
  消除NGSIM原始数据中的定位抖动问题，确保速度、加速度计算结果符合物理规律

处理方法：
  (1) 采用 Savitzky-Golay 滤波器进行平滑处理
  (2) 滤波器窗口参数需根据数据特点进行调优
  (3) 平滑后需验证加速度值是否在合理物理范围内（通常为 ±3 m/s²）

输出要求：
  [ ] 生成可视化对比图像（原始数据 vs 平滑后数据）
  [ ] 生成Markdown格式分析报告，保存至 ./doc/
  [ ] 备份平滑去噪后的数据至独立文件

4.2 原始数据清洗与坐标转换
--------------------------
处理内容：
  (1) 坐标对齐：统一NGSIM原始数据的Local_Y（纵向位置）方向
  (2) 单位换算：
      - 长度单位：英尺(feet) → 米(m)，转换比例 1 feet = 0.3048 m
      - 速度单位：英里/小时(mph) → 米/秒(m/s)，转换比例 1 mph = 0.44704 m/s
      - 加速度单位：ft/s² → m/s²
  (3) 异常值清洗：剔除速度、加速度明显异常的数据点

输出要求：
  [ ] 生成坐标转换前后对比可视化图像
  [ ] 生成单位换算说明文档
  [ ] 生成Markdown格式分析报告，保存至 ./doc/

4.3 跟驰对（Car-following Pairs）提取
-------------------------------------
匹配规则：
  (1) 唯一ID匹配：根据Vehicle_ID和Preceding_ID提取前后车配对关系
  (2) 时空连续性检查：
      - 前后车必须位于同一车道（Lane_ID相同）
      - 共存时间需超过设定阈值（建议15秒以上）
      - 相邻帧之间的车辆配对关系应保持稳定
  (3) 异常剔除：
      - 过滤掉存在换道行为（Lane Change）的片段
      - 剔除前后车距离出现突变的数据

输出要求：
  [ ] 生成跟驰对数量分布可视化图像
  [ ] 生成跟驰持续时间统计图
  [ ] 生成Markdown格式分析报告，保存至 ./doc/

4.4 特征变量计算
----------------
计算时刻：每一时刻（逐帧计算）

特征定义：
  (1) 后车速度 (v_f)：直接提取平滑处理后的后车速度数据
  (2) 相对距离 (s)：s = y_leader - y_follower - L_leader
      说明：
        - y_leader：前车纵向位置
        - y_follower：后车纵向位置
        - L_leader：前车车长
        - 计算结果为前后车车身之间的净间距（非车头间距）
  (3) 相对速度 (v_rel)：v_rel = v_leader - v_follower
      说明：正值表示前车速度快于后车（后车正在追赶）
  (4) 标签（Label）：后车的加速度

输出要求：
  [ ] 生成各特征变量的分布统计图
  [ ] 生成特征变量间相关性分析图
  [ ] 生成Markdown格式分析报告，保存至 ./doc/

4.5 数据标准化与分段
--------------------
处理方法：
  (1) 滑动窗口切片：使用滑动窗口将长轨迹切割成固定长度的时间序列片段
  (2) 窗口重叠：相邻窗口之间可设置一定重叠比例（如50%），以增加样本数量
  (3) 版本要求：
      - 版本一：10秒为一个clip
      - 版本二：20秒为一个clip
  (4) 切片筛选：剔除长度不足的片段

输出要求：
  [ ] 生成切片长度分布可视化图像
  [ ] 生成数据切分统计报告
  [ ] 生成Markdown格式分析报告，保存至 ./doc/

4.6 数据持久化
--------------
输出格式：HDF5 (.h5)

数据存储结构（建议）：
  ├── train_10s.h5          # 10秒训练集
  ├── test_10s.h5           # 10秒测试集
  ├── train_20s.h5          # 20秒训练集
  └── test_20s.h5           # 20秒测试集

H5文件内部数据结构建议：
  ├── x                     # 输入特征 (shape: [n_samples, seq_len, n_features])
  ├── y                     # 标签 (shape: [n_samples, seq_len])
  └── metadata              # 元数据（可选）

4.7 数据集可视化分析
--------------------
分析内容：
  (1) 数据基本统计：样本数量、特征分布、缺失值检查
  (2) 特征工程分析：各特征的分布形态、异常值检测
  (3) 轨迹可视化：选取典型样本展示车辆运动轨迹
  (4) 跟驰行为分析：跟驰对的空间分布、速度分布

输出要求：
  [ ] 生成多样化可视化图像（直方图、散点图、热力图等）
  [ ] 生成综合数据分析Markdown报告，保存至 ./doc/

4.8 模型验证
------------
验证方法：
  (1) 构建神经网络模型进行验证
  (2) 优先使用LSTM简易模型进行功能验证
  (3) 评估指标：MSE、MAE、R²等回归指标
  (4) 分析模型预测误差的分布规律

输出要求：
  [ ] 生成模型训练曲线（loss曲线、验证曲线）
  [ ] 生成预测结果对比图
  [ ] 生成模型性能评估Markdown报告，保存至 ./doc/
